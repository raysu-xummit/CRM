# 醫美客戶 CRM 數據儀表板

> 開始時間：2026-03-12

## Why（為什麼做）

集團旗下多家醫美診所已累積約 60 萬筆客戶資料，但目前缺乏視覺化的數據呈現工具。管理層與前線人員無法直觀地掌握客戶行為、營運狀況與行銷成效，也不清楚 CRM 應該關注哪些關鍵指標。

## Goal（目標）

建立一個 CRM 數據儀表板（Dashboard），將既有客戶資料轉化為可視化的洞察。讓不同角色（總公司行銷、總店長、各店店長、Sales）各自看到該關心的指標，支持數據驅動的決策。

**注意：這不是建立新的管理系統，而是對既有資料的呈現與分析。**

## Background（背景）

- **組織架構**：總公司 + 旗下多家醫美診所
- **使用者角色**：
  - 總公司行銷部門：全局客群分析、行銷活動 ROI
  - 總店長：跨門市對比、整體營運監控
  - 各店店長：單店營運、團隊績效、客戶健康度
  - Sales（前線）：個人客戶 360° 視圖、業績追蹤
- **資料規模**：約 60 萬客戶
- **資料來源**：既有系統（資料結構待確認）
- **技術棧**：C# .NET Core Web API + MSSQL + React TypeScript

## 權限劃分

| 角色 | 可見範圍 |
|------|----------|
| 總公司行銷 | 全部門市、全部客戶、行銷分析 |
| 總店長 | 全部門市營運數據 |
| 各店店長 | 自己門市的數據 |
| Sales | 自己負責的客戶 |
