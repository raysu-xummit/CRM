# 醫美 CRM 指標體系

> 從宏觀到微觀的六層指標架構

```
集團全局 → 門市對比 → 客群分層 → 單店營運 → 客戶個人 → 單筆交易
  ↑ 戰略決策                                          實際執行 ↓
```

高層往下看趨勢與結構，前線往上看具體客戶與行動。
好的 CRM Dashboard 是讓每個角色打開就看到自己該關心的東西。

---

## Level 1｜集團全局（總公司視角）

> 回答：**整個集團的生意健不健康？往哪個方向走？**

| 指標 | 說明 |
|------|------|
| 總營收 & 成長率 | 月/季/年營收趨勢，同期比較 |
| 活躍客戶率 | 60 萬客戶中，近 N 個月有消費的佔多少？ |
| 新客獲取數 & 成本 | 每月新增多少客戶，每個新客的取得成本 |
| 整體留存率 / 流失率 | 客戶有沒有持續回來 |
| 客戶集中度風險 | 營收是否過度依賴頭部客戶（Top 10% 貢獻多少 %） |
| 各門市營收佔比 | 哪些店撐起業績、哪些店需要關注 |
| 療程大類營收結構 | 光電類 vs 注射類 vs 保養類的營收比例與趨勢 |

---

## Level 2｜門市對比（總店長視角）

> 回答：**各門市表現如何？誰好誰差？差在哪？**

| 指標 | 說明 |
|------|------|
| 門市營收排行 & 趨勢 | 各店月營收，看趨勢而不只看絕對值 |
| 門市客單價比較 | 同樣的客人在不同店消費力不同嗎？ |
| 新客 vs 回頭客比例 | 某店全靠新客撐 → 留不住人的警訊 |
| 門市回購率比較 | 哪家店的客人最忠誠 |
| 人均產值 | 每個 Sales 平均貢獻營收，衡量門市效率 |
| 熱門療程差異 | 各店暢銷項目不同 → 反映客群或 Sales 推薦策略差異 |
| 跨店消費比例 | 有多少客人會去不同門市，跨店流動的模式 |

---

## Level 3｜客群分層（行銷部門視角）

> 回答：**我的客戶是誰？怎麼分群？對誰做什麼事？**

| 指標 | 說明 |
|------|------|
| RFM 分群分佈 | 將客戶分成 VIP / 高潛力 / 需喚醒 / 已流失等群組 |
| 客戶生命週期分佈 | 新客 → 活躍 → 沉睡 → 流失，各階段人數與佔比 |
| 階段遷移分析 | 這個月有多少人從「活躍」變「沉睡」？多少從「沉睡」被喚醒？ |
| 各群組 LTV | VIP 客戶的 LTV 是普通客戶的幾倍？ |
| 客群輪廓 | 年齡、性別、地區分佈 → 行銷投放依據 |
| 療程偏好矩陣 | 不同客群偏好什麼療程 → 交叉銷售的基礎 |
| 價格敏感度 | 哪群人對折扣有反應、哪群人在意品質不在意價格 |
| 新客轉熟客率 | 首次消費後 90 天內再次消費的比例，最關鍵的轉化點 |

---

## Level 4｜單店營運（各店店長視角）

> 回答：**我這家店的狀況怎麼樣？我的團隊表現如何？**

| 指標 | 說明 |
|------|------|
| 本店營收 & 目標達成率 | 對比目標的完成進度 |
| Sales 業績排行 | 個人業績、成交筆數、客單價 |
| 本店客戶健康度 | 活躍 / 沉睡 / 流失的客戶比例 |
| 回訪率 | 上個月來的客人，這個月回來多少 |
| 流失預警名單 | 超過 X 天沒回來的客人清單，可以主動聯繫 |
| 本店熱門 / 冷門療程 | 哪些項目賣得好、哪些該調整推廣策略 |
| 預約狀況 | 預約滿載率、no-show 率 |

---

## Level 5｜個人客戶 360°（Sales 視角）

> 回答：**眼前這個客人，我該怎麼服務她？**

| 指標 | 說明 |
|------|------|
| 基本資料 | 姓名、年齡、聯絡方式、過敏/禁忌 |
| 消費時間軸 | 每次來做了什麼、花了多少、誰服務的 |
| 累計消費 & LTV | 這個客人的總價值 |
| 消費趨勢 | 花越來越多？還是越來越少？ |
| 到店頻率 & 週期 | 平均多久來一次 → 預測下次應該什麼時候來 |
| 偏好療程 | 她最常做什麼 → 推薦相關項目 |
| 最後到店距今天數 | 超過平均週期 → 該主動聯繫了 |
| 儲值 / 方案餘額 | 還有沒有用完的療程或儲值金 |

---

## Level 6｜單筆交易（最微觀）

> 回答：**這筆生意的細節**

| 指標 | 說明 |
|------|------|
| 交易日期、門市、Sales | 基本事實 |
| 療程項目明細 | 做了什麼、數量、單價 |
| 實收金額 vs 定價 | 折扣了多少 → 折扣率 |
| 付款方式 | 現金 / 刷卡 / 儲值扣款 |
| 是否為新客首次消費 | 標註新客轉化 |
