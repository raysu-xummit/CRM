# Level 3｜客群分層指標 — 詳細定義

> 目標受眾：總公司行銷部門
> 核心問題：我的客戶是誰？怎麼分群？對誰做什麼事？

---

## 3.1 RFM 分群分佈

### 定義
基於 Recency（最近消費）、Frequency（消費頻率）、Monetary（消費金額）三個維度，對所有客戶進行分群，識別不同價值等級的客戶。

### 計算參數

| 參數 | 定義 | 資料來源欄位 |
|------|------|-------------|
| R (Recency) | 客戶最後一次消費距今天數 | `交易日期` → `MAX(交易日期)` → `DATEDIFF(天, MAX(交易日期), TODAY)` |
| F (Frequency) | 統計期間內的消費次數（以「到店次」計，同一天多筆算一次） | `交易日期` → `COUNT(DISTINCT 交易日期)` |
| M (Monetary) | 統計期間內的累計消費金額 | `實收金額` → `SUM(實收金額)` |

### 計算方式

1. **統計期間**：近 12 個月（可調整）
2. **評分方式**：每個維度依百分位分為 5 等級（1-5 分）
   - R：天數越少分越高（最近才來 = 5 分）
   - F：次數越多分越高
   - M：金額越大分越高
3. **分群邏輯**：

| 客群名稱 | R 分數 | F 分數 | M 分數 | 說明 |
|----------|--------|--------|--------|------|
| 重要價值客戶 (VIP) | 4-5 | 4-5 | 4-5 | 最近來、常來、花很多 |
| 重要發展客戶 | 4-5 | 1-3 | 4-5 | 最近來、花很多、但不常來 → 提高頻次 |
| 重要保持客戶 | 1-3 | 4-5 | 4-5 | 常來花很多、但最近沒來 → 預防流失 |
| 重要挽留客戶 | 1-2 | 1-3 | 4-5 | 以前花很多、但很久沒來了 → 緊急挽回 |
| 一般價值客戶 | 3-5 | 3-5 | 1-3 | 活躍但消費力一般 → 提升客單價 |
| 一般發展客戶 | 4-5 | 1-2 | 1-3 | 最近來但消費少又不常來 → 多為新客 |
| 一般保持客戶 | 1-3 | 3-5 | 1-3 | 以前常來但最近沒來、消費一般 |
| 流失客戶 | 1-2 | 1-2 | 1-3 | 很久沒來、不常來、花不多 |

### Dashboard 呈現
- RFM 分群客戶數量分佈圖（長條圖 / 圓餅圖）
- 各群組的平均 R、F、M 值
- 各群組佔總營收的百分比
- 時間趨勢：各群組人數月度變化

### 所需資料欄位
- `客戶ID`、`交易日期`、`實收金額`、`門市ID`

## 3.2 客戶生命週期分佈

### 定義
將客戶依其消費行為劃分至生命週期的不同階段，掌握整體客戶結構的健康程度。

### 計算參數

| 參數 | 定義 |
|------|------|
| 首次消費日期 | `MIN(交易日期)` per 客戶 |
| 最後消費日期 | `MAX(交易日期)` per 客戶 |
| 距今天數 | `DATEDIFF(天, MAX(交易日期), TODAY)` |
| 消費次數 | `COUNT(DISTINCT 交易日期)` per 客戶 |
| 客戶年齡（天） | `DATEDIFF(天, MIN(交易日期), TODAY)` |

### 生命週期階段定義

| 階段 | 條件 | 說明 |
|------|------|------|
| **新客** | 首次消費在近 90 天內 AND 消費次數 = 1 | 剛來第一次，尚未回購 |
| **活躍客戶** | 最後消費在近 90 天內 AND 消費次數 ≥ 2 | 有回購行為且近期活躍 |
| **沉睡客戶** | 最後消費在 91-180 天前 | 有一陣子沒來了 |
| **流失風險客戶** | 最後消費在 181-365 天前 | 快要流失 |
| **已流失客戶** | 最後消費超過 365 天 | 一年以上未消費 |

> **注意**：天數門檻（90/180/365）需根據醫美產業的實際回訪週期調整。可先以此為預設，上線後依數據校正。

### Dashboard 呈現
- 各階段客戶人數與佔比（漏斗圖或堆疊長條圖）
- 月度趨勢：各階段人數隨時間的變化
- 各階段客戶的平均 LTV

### 所需資料欄位
- `客戶ID`、`交易日期`

## 3.3 階段遷移分析

### 定義
追蹤客戶在不同生命週期階段之間的流動，了解客戶是在進步（留存/喚醒）還是退步（沉睡/流失）。

### 計算參數

| 參數 | 定義 |
|------|------|
| 本期階段 | 以本月最後一天為基準，依 3.2 規則計算的生命週期階段 |
| 上期階段 | 以上月最後一天為基準，依 3.2 規則計算的生命週期階段 |
| 遷移方向 | 本期階段 vs 上期階段的變化 |

### 遷移矩陣

```
            本期 →
上期 ↓      新客    活躍    沉睡    流失風險    已流失
新客         -      升級     惡化     惡化       惡化
活躍        --      維持     惡化     惡化       惡化
沉睡        --      喚醒     維持     惡化       惡化
流失風險    --      喚醒     改善     維持       惡化
已流失      --      回歸     回歸     --         維持
```

### 關鍵遷移指標

| 指標 | 計算方式 | 意義 |
|------|----------|------|
| 新客轉活躍率 | 上期「新客」本期變「活躍」的人數 / 上期「新客」總人數 | 新客有沒有回來 |
| 活躍流失率 | 上期「活躍」本期變「沉睡」或更差的人數 / 上期「活躍」總人數 | 活躍客戶的維繫能力 |
| 沉睡喚醒率 | 上期「沉睡」本期變「活躍」的人數 / 上期「沉睡」總人數 | 喚醒行銷的效果 |
| 流失回歸率 | 上期「已流失」本期變「活躍」的人數 / 上期「已流失」總人數 | 挽回能力 |
| 淨遷移方向 | (正向遷移人數 - 負向遷移人數) / 總客戶數 | 整體客戶基盤是變好還是變差 |

### Dashboard 呈現
- 桑基圖 (Sankey Diagram)：從上期到本期的客戶流動
- 月度遷移率趨勢折線圖
- 重點警示：當負向遷移明顯大於正向時 highlight

### 所需資料欄位
- `客戶ID`、`交易日期`（需要歷史月度快照或可回溯計算）

## 3.4 各群組 LTV（客戶終身價值）

### 定義
計算不同客群的平均客戶終身價值，量化各群組的商業貢獻差異。

### 計算參數

| 參數 | 定義 | 計算方式 |
|------|------|----------|
| 歷史 LTV | 客戶從首次消費至今的累計實收金額 | `SUM(實收金額)` per 客戶 |
| 平均月消費 | 客戶活躍月份的平均月消費額 | `SUM(實收金額)` / `活躍月份數` |
| 活躍月份數 | 有消費行為的月份數量 | `COUNT(DISTINCT YEAR_MONTH(交易日期))` |
| 客戶壽命（月） | 首次到最後消費的月份跨度 | `DATEDIFF(月, MIN(交易日期), MAX(交易日期))` |
| 預測 LTV | 預估未來 12 個月可帶來的營收 | `平均月消費 × 預測留存月數` |

### 群組對比維度

| 對比方式 | 說明 |
|----------|------|
| 按 RFM 群組 | VIP 的 LTV 是流失客戶的幾倍？ |
| 按生命週期 | 活躍客戶 vs 沉睡客戶的 LTV 差距 |
| 按首次消費療程 | 第一次做雷射的客人 LTV 比做保養的高嗎？ |
| 按獲取來源 | 轉介紹客戶 vs 廣告客戶的 LTV |

### Dashboard 呈現
- 各群組平均 LTV 長條圖對比
- LTV 分佈直方圖（了解離散程度）
- LTV Top 100 客戶名單
- 群組 LTV 月度趨勢

### 所需資料欄位
- `客戶ID`、`交易日期`、`實收金額`、`療程項目`、`獲客來源`（若有）

## 3.5 客群輪廓

### 定義
描繪各客群的人口統計特徵與行為特徵，建立客戶畫像，作為行銷策略和投放的依據。

### 計算參數

| 維度 | 參數 | 計算方式 |
|------|------|----------|
| 人口統計 | 年齡分佈 | `YEAR(TODAY) - 出生年` → 分群 (20-29, 30-39, 40-49, 50+) |
| | 性別比例 | `COUNT` by 性別 |
| | 地區分佈 | 依 `地址` 或 `門市所在區域` 分群 |
| 行為特徵 | 平均客單價 | `AVG(單次消費金額)` per 群組 |
| | 平均到店頻率 | `AVG(年消費次數)` per 群組 |
| | 偏好消費時段 | `MODE(交易時段)` — 平日/週末、上午/下午/晚上 |
| | 偏好門市 | `MODE(門市ID)` per 客戶 |

### Dashboard 呈現
- 客群輪廓卡片：每個 RFM 群組一張卡片，顯示該群組的典型特徵
- 年齡 × 消費金額 散佈圖
- 地區熱力圖
- 各群組特徵雷達圖對比

### 所需資料欄位
- `客戶ID`、`性別`、`出生日期`、`地址/區域`、`交易日期`、`交易時間`、`實收金額`、`門市ID`

## 3.6 療程偏好矩陣

### 定義
分析不同客群對不同療程類別的偏好程度，識別交叉銷售（Cross-sell）與向上銷售（Up-sell）機會。

### 計算參數

| 參數 | 定義 | 計算方式 |
|------|------|----------|
| 療程大類 | 療程的分類（如：光電類、注射類、保養類、體雕類） | 依 `療程項目` 對應到 `療程大類` |
| 群組滲透率 | 某群組中購買過該療程大類的客戶佔比 | `COUNT(DISTINCT 客戶ID WHERE 療程大類=X)` / `群組總客戶數` |
| 群組貢獻度 | 某群組在某療程大類的營收佔該療程總營收的比例 | `SUM(群組在該療程的實收)` / `SUM(該療程總實收)` |
| 跨類購買率 | 購買過 2 種以上療程大類的客戶佔比 | `COUNT(客戶 HAVING COUNT(DISTINCT 療程大類) ≥ 2)` / `總客戶數` |
| 療程組合關聯 | 常見的療程搭配組合 | 關聯分析（A 療程客戶中有 X% 也做了 B 療程） |

### 偏好矩陣範例

```
              光電類    注射類    保養類    體雕類
VIP            85%      72%      60%      45%
高潛力客戶      55%      40%      70%      20%
一般客戶        30%      15%      80%      10%
沉睡客戶        40%      25%      50%       8%
```
> 數字 = 群組滲透率（該群組中有多少 % 的人購買過此療程大類）

### Dashboard 呈現
- 熱力圖：客群 × 療程大類的滲透率矩陣
- 療程組合桑基圖：客戶從療程 A 到療程 B 的流動
- 交叉銷售推薦：「購買 A 的客戶有 X% 也購買了 B」

### 所需資料欄位
- `客戶ID`、`療程項目`、`療程大類`、`實收金額`、`交易日期`

## 3.7 價格敏感度

### 定義
分析不同客群對價格變動和促銷活動的反應程度，協助制定差異化定價和行銷策略。

### 計算參數

| 參數 | 定義 | 計算方式 |
|------|------|----------|
| 平均折扣率 | 客戶消費的平均折扣幅度 | `AVG(1 - 實收金額 / 定價金額)` per 客戶 |
| 促銷參與率 | 客戶消費中有多少次是在促銷期間 | `COUNT(促銷期間交易)` / `COUNT(全部交易)` |
| 促銷提升倍數 | 促銷期間消費金額 vs 非促銷期間 | `AVG(促銷期消費)` / `AVG(非促銷期消費)` |
| 價格帶偏好 | 客戶主要消費的價格區間 | 依單次消費金額分帶：低 (<3K) / 中 (3K-10K) / 高 (10K-30K) / 頂級 (30K+) |

### 敏感度分群

| 群組 | 特徵 | 行銷策略 |
|------|------|----------|
| 價格驅動型 | 高折扣率 + 高促銷參與率 | 用優惠吸引，但控制折扣深度 |
| 價值導向型 | 低折扣率 + 低促銷參與率 + 高客單價 | 強調品質和服務，不需要打折 |
| 機會型 | 促銷時大量消費，非促銷時極少消費 | 定期觸發但避免過度依賴 |
| 忠誠型 | 不論促銷與否都穩定消費 | VIP 專屬服務，維持關係 |

### Dashboard 呈現
- 各敏感度群組的人數與營收佔比
- 促銷 vs 非促銷期間的營收對比趨勢
- 折扣率 × 消費頻次 散佈圖

### 所需資料欄位
- `客戶ID`、`交易日期`、`實收金額`、`定價金額`（原價）、`是否促銷期間`（或可從活動日期推算）

## 3.8 新客轉熟客率

### 定義
追蹤首次消費客戶在一定期間內是否產生二次消費，是醫美 CRM 最關鍵的轉化指標之一。

### 計算參數

| 參數 | 定義 | 計算方式 |
|------|------|----------|
| 首次消費日期 | 客戶第一次交易日期 | `MIN(交易日期)` per 客戶 |
| 二次消費日期 | 客戶第二次交易日期 | 第二早的 `交易日期`（不同天） |
| 轉化天數 | 首次到二次消費的間隔 | `DATEDIFF(天, 首次消費日期, 二次消費日期)` |
| N 天轉化率 | 首次消費後 N 天內有二次消費的比例 | `COUNT(轉化天數 ≤ N 的客戶)` / `COUNT(首次消費在觀察期起始的客戶)` |

### 關鍵轉化率指標

| 指標 | 計算方式 | 建議觀察值 |
|------|----------|-----------|
| 30 天轉化率 | 首次消費後 30 天內回購的比例 | 高轉化，通常因療程需回診 |
| 60 天轉化率 | 首次消費後 60 天內回購的比例 | 中等轉化 |
| 90 天轉化率 | 首次消費後 90 天內回購的比例 | 作為「是否轉化為熟客」的主要判斷線 |
| 180 天轉化率 | 首次消費後 180 天內回購的比例 | 長尾轉化的最終觀察 |

### 分析切角

| 維度 | 說明 |
|------|------|
| 按首次療程 | 第一次做什麼療程的客人最容易回來？ |
| 按門市 | 哪家門市的新客轉化最好？ |
| 按 Sales | 哪位 Sales 服務的新客最容易回來？ |
| 按獲客來源 | 轉介紹的新客 vs 廣告的新客，轉化率差多少？ |
| 按月份 | 轉化率有沒有季節性波動？ |

### Dashboard 呈現
- 整體 30/60/90/180 天轉化率數字卡片
- 各月份新客 cohort 的轉化率追蹤曲線（Cohort Analysis）
- 按門市 / Sales / 療程的轉化率對比長條圖
- 轉化天數分佈直方圖：大部分人幾天後回來？

### 所需資料欄位
- `客戶ID`、`交易日期`、`門市ID`、`Sales ID`、`療程項目`、`獲客來源`（若有）

## 資料欄位需求彙總

以下為 Level 3 所有指標所需的資料欄位彙總：

### 必要欄位（缺一不可）

| 欄位 | 用途 | 使用指標 |
|------|------|----------|
| `客戶ID` | 客戶唯一識別 | 全部 |
| `交易日期` | 消費時間 | 全部 |
| `實收金額` | 實際收取金額 | 3.1, 3.4, 3.5, 3.6, 3.7 |
| `門市ID` | 所屬門市 | 3.5, 3.8 |
| `療程項目` | 具體療程名稱 | 3.4, 3.6 |

### 重要欄位（強烈建議有）

| 欄位 | 用途 | 使用指標 |
|------|------|----------|
| `療程大類` | 療程分類 | 3.6 |
| `定價金額` | 原價（計算折扣率） | 3.7 |
| `Sales ID` | 服務人員 | 3.8 |
| `性別` | 客群輪廓 | 3.5 |
| `出生日期` | 年齡計算 | 3.5 |

### 加分欄位（有的話更好）

| 欄位 | 用途 | 使用指標 |
|------|------|----------|
| `獲客來源` | 客戶從哪來 | 3.4, 3.8 |
| `地址/區域` | 地理分佈 | 3.5 |
| `促銷活動ID` | 是否在促銷期間消費 | 3.7 |
| `交易時間` | 消費時段偏好 | 3.5 |
